Template engine.

Django ships with its own template engine (DTL). A template is a text file mixing static content with special constructs that are evaluated at render time against a context (a dictionary of variables).

Blocks

{% verbatim %}{% block name %}...{% endblock %}{% endverbatim %} defines a region a child template can override. Combined with {% verbatim %}{% extends %}{% endverbatim %} it powers template inheritance and the DRY principle: this very page overrides the style block to load a different stylesheet.

For loops

{% verbatim %}{% for item in list %} {{ item }} {% endfor %}{% endverbatim %} iterates over any iterable passed in the context.

If control structures

{% verbatim %}{% if condition %}...{% elif other %}...{% else %}...{% endif %}{% endverbatim %} renders content conditionally.

Displaying context variables

A variable is printed with double curly braces: {% verbatim %}{{ variable }}{% endverbatim %}, and filters transform it: {% verbatim %}{{ name|upper }}{% endverbatim %}.
